## ⚠️ 人工验证需求（浏览器控制服务不可用）

由于 OpenClaw 浏览器控制服务连接问题，以下项目需要**人工验证**：

### 必须验证的GSAP动画（7项）

#### 1. 首页入场动画（animatePageEntry）
**预期效果：**
- Header 从上方淡入（y: -20 → 0）
- Hero 标题渐进式展开（stagger 0.15s）
- 搜索框弹出（scale: 0.95 → 1）
- 统计卡片依次出现（stagger 0.1s）
- 教授列表最后淡入

**验证方法：**
1. 打开 https://ohmyprofessors.com
2. 观察页面加载后的入场动画
3. 确认元素按顺序出现，有流畅的渐进感

#### 2. 教授卡片悬停动画（setupProfessorCardHover）
**预期效果：**
- 鼠标悬停：卡片上移 8px
- 阴影增强：`0 12px 24px rgba(0,0,0,0.15)`
- 动画时长：0.3s
- 鼠标移出：平滑恢复

**验证方法：**
1. 在首页找到任意教授卡片
2. 鼠标悬停
3. 确认卡片有"浮起"效果

#### 3. 教授列表Stagger加载（animateProfessorList）
**预期效果：**
- 卡片依次从下方淡入（y: 20 → 0）
- 每个卡片延迟 0.08s
- 有"波浪式"出现的视觉节奏

**验证方法：**
1. 首页加载完成后
2. 观察教授卡片列表
3. 确认卡片不是同时出现，而是有节奏地依次出现

#### 4. 搜索结果更新动画（animateSearchResults）
**预期效果：**
- 旧内容淡出并上移（opacity: 1 → 0, y: 0 → -10）
- 新内容淡入并复位（opacity: 0 → 1, y: -10 → 0）
- 平滑过渡，无突兀切换

**验证方法：**
1. 在搜索框输入"Sarah"
2. 观察卡片列表更新时的过渡动画
3. 确认有淡入淡出效果

#### 5. 教授详情页入场动画（animateProfessorDetailEntry）
**预期效果：**
- 返回按钮从左侧滑入（x: -20 → 0）
- 教授信息头部从下方淡入（y: 30 → 0）
- 统计卡片依次缩放入场（scale: 0.9 → 1, stagger 0.1s）
- 评价列表淡入
- CTA按钮最后出现

**验证方法：**
1. 点击任意教授卡片
2. 观察详情页加载动画
3. 确认元素按顺序出现

#### 6. 评价卡片Scroll Reveal（setupReviewScrollReveal）
**预期效果：**
- 滚动时，评价卡片从下方淡入（y: 30 → 0）
- 触发点：卡片进入视口底部100px时
- 有延迟效果（delay: index * 0.05s）

**验证方法：**
1. 进入有多个评价的教授详情页（当前无评价，Phase 2实现）
2. 滚动页面
3. 确认评价卡片逐渐出现

#### 7. CTA按钮脉冲动画（setupCTAPulse）
**预期效果：**
- 按钮轻微缩放（scale: 1 → 1.05 → 1）
- 循环播放，速度1.5s
- Ease: sine.inOut（平滑呼吸感）

**验证方法：**
1. 观察首页底部的"Write a Review"按钮
2. 或教授详情页的"Write the First Review"按钮
3. 确认按钮有轻微的"呼吸"效果
