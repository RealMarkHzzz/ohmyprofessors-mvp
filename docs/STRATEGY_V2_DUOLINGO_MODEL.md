## 战略转向声明 (Strategic Pivot Declaration)

### 核心问题识别

**旧模型的根本缺陷：**
```
用户行为路径：
选课前 → 打开网站 → 搜索教授 → 读评价 → 做决定 → 离开
                                              ↓
                                        4个月后再来
```

**致命问题：**
- 使用频率：每学期2次（选课周）
- 停留时长：5-10分钟
- 生命周期：4年（本科期间）
- **结果：极低的DAU/MAU比（预估<5%），无法建立习惯**

### 新模型的核心转向

**From → To 转变矩阵：**

| 维度 | 旧模型 (v1.0) | 新模型 (v2.0) |
|------|--------------|--------------|
| **使用目的** | 一次性查询 | 持续学习伴侣 |
| **频率设计** | 被动等待（学期2次） | 主动引导（每日/每周） |
| **价值感知** | 工具性（Get info & Leave） | 陪伴性（Always helping） |
| **数据输入** | 仅在有需求时贡献 | 贡献本身成为游戏 |
| **奖励机制** | 无 | 即时反馈 + 长期成就 |
| **社交属性** | 单向阅读 | 双向互动 + 社区归属 |

## 一、核心原则重构 (Core Principles Reconstruction)

### 1.1 战略北极星转移

**Old North Star:**  
每周新增高质量评价数 (Weekly Quality Reviews)

**New North Star:**  
**每日活跃用户的平均交互次数** (Daily Active User Engagement Frequency)

**为什么？**
- Duolingo 的成功不在于"学了多少"，而在于"每天都来"
- 高频交互 = 习惯养成 = 长期留存 = 网络效应
- 从"需求驱动"转向"习惯驱动"

### 1.2 产品定位重构

**旧定位：**  
> 教授评价查询平台（Query-based Tool）

**新定位：**  
> 大学选课能力培养系统（Skill-building Companion）

**Tagline 演变：**
- Old: "Rate your professor, Get GPA"  
- New: "Master uni, one class at a time" (掌握大学，一节课一节课来)

### 1.3 用户心智模型转变

**从"我需要什么" → "我在进步"**

```
旧心智：
我要选COMP1012 → 搜索教授 → 读评价 → 选好了 → 离开

新心智：
我在规划我的课程路径 → 
今天解锁了"CS入门"模块 → 
完成了3门课程的难度评估挑战 → 
获得"选课策略家Lv.2"徽章 → 
明天继续学习"GPA优化技巧"
```

## 二、Duolingo 机制深度迁移 (Mechanism Mapping)

### 2.1 任务碎片化架构 (Micro-Action Design)

#### 【机制原理】
Duolingo 不让你"学西班牙语"，而是让你"完成5分钟的今日课程"。

#### 【迁移到选课场景】

**核心转变：**  
不让用户"评价教授"，而是让用户"完成今日选课任务"。

**微任务拆解（Micro-Actions）：**

| 任务类型 | 时长 | 奖励 | 示例 |
|---------|------|------|------|
| **快速点评** | 30秒 | +5 XP | "这门课考试是Open Book吗？" (单选题) |
| **难度评估** | 1分钟 | +10 XP | "对比COMP1012 vs COMP1015的难度" (滑动条) |
| **经验分享** | 2分钟 | +20 XP | "分享一个这门课的学习技巧" (50字) |
| **课程规划** | 3分钟 | +30 XP | "规划下学期的3门课组合" (拖拽排序) |
| **帮助新生** | 1分钟 | +15 XP | "回答1个新生的选课问题" (Q&A) |

**关键设计：**
- 每个任务都有"进度感"（1/5 完成）
- 每个任务都有"即时奖励"（XP + 成就）
- 每个任务都可"中断续做"（保存草稿）

### 2.2 连续关卡式推进 (Streak \u0026 Leveling System)

#### 【机制原理】
Duolingo 的"连续打卡"让你害怕失去已投入的时间成本。

#### 【迁移到选课场景】

**选课能力地图（Course Planning Skill Tree）：**

```
Level 1: 新生入门
├─ 解锁: 基础课程浏览
├─ 任务: 浏览10门课程简介 ✓
├─ 任务: 完成1次课程难度评估 ✓
└─ 奖励: 解锁"课程收藏"功能

Level 2: 选课探索者
├─ 解锁: 高级筛选功能
├─ 任务: 对比3门课程的难度和工作量
├─ 任务: 收藏5门感兴趣的课程
└─ 奖励: 解锁"AI课程推荐"

Level 3: 策略规划师
├─ 解锁: 学期规划工具
├─ 任务: 规划一个完整的学期课表
├─ 任务: 获得3个同学点赞
└─ 奖励: 解锁"GPA模拟器"

Level 4: 选课导师
├─ 解锁: 社区答疑权限
├─ 任务: 帮助5个新生解答问题
├─ 任务: 分享3个高质量学习技巧
└─ 奖励: 专属"导师"徽章 + 优先推荐位

Level 5: 大学精通者
├─ 解锁: 全校课程数据洞察
├─ 任务: 贡献10条深度课程评价
├─ 任务: 连续30天打卡
└─ 奖励: Premium功能永久免费
```

**Streak 机制设计：**
- **每日目标**：完成1个微任务（5分钟）
- **Streak 奖励**：
  - 7天 → "坚持者"徽章 + 2x XP加成
  - 14天 → 解锁"课程趋势"功能
  - 30天 → Premium试用30天
  - 100天 → 进入"传奇用户"榜单

**Streak 保护机制：**
- 允许1次"冻结日"（花费100 XP购买）
- Streak中断后，提供"修复任务"（完成3个微任务）

### 2.3 即时反馈与正向强化 (Instant Gratification)

#### 【机制原理】
Duolingo 每答对一道题都有音效、动画、鼓励文案。

#### 【迁移到选课场景】

**交互反馈设计（Feedback Loops）：**

| 用户行为 | 即时反馈 | 延迟反馈 |
|---------|---------|---------|
| 浏览课程 | "太好了，你已经看了3门课程了！" | +5 XP，进度条+10% |
| 评估难度 | "你的评估帮助了12个同学！" | +10 XP，解锁"评估达人"成就 |
| 分享技巧 | "你的技巧被点赞了！" | +15 XP，提升"社区贡献度" |
| 完成规划 | "完美的学期规划！预计GPA 6.5" | +30 XP，解锁下一关卡 |
| 帮助新生 | "你刚刚解决了新生的困惑！" | +20 XP，"导师"徽章进度+1 |

**拟人化反馈文案库：**
- 成功时："太棒了！你刚刚帮助了X个同学做决定"
- 进步时："你已经连续X天在进步了，继续加油！"
- 接近目标时："只差1步就能解锁新功能了！"
- 中断后回归："欢迎回来！你的朋友们都在等你"

**错误纠正而非惩罚：**
- 不说"你的评价太短了"
- 改说"再补充10个字，会帮助更多同学哦"

### 2.4 可视化进度与成长感 (Progress Visualization)

#### 【机制原理】
Duolingo 的语言树、经验值、徽章让你看到自己"变强了"。

#### 【迁移到选课场景】

**1. 选课能力雷达图（Skill Radar）**

```
        评估能力
           /\
          /  \
   规划能力    判断能力
        /      \
       /        \
  社区贡献 ---- 数据洞察
```

每完成任务，对应能力值+1，雷达图扩大。

**2. 课程收集册（Course Collection）**

类似Duolingo的"单词本"，用户收集的课程卡片：

```
[已探索] Computer Science (12/50门)
├─ COMP1012 ⭐⭐⭐⭐⭐
├─ COMP1015 ⭐⭐⭐⭐
├─ COMP2013 🔒 (需解锁Level 2)
└─ ...

[已探索] Mathematics (5/30门)
[未探索] Physics (0/25门)
```

**3. 成就墙（Achievement Wall）**

- 🏆 **选课新手**：完成首个课程评估
- 🎯 **策略家**：规划3个不同难度组合
- 💡 **导师之星**：被点赞超过100次
- 🔥 **不灭火焰**：连续打卡30天
- 📊 **数据控**：查看超过50门课程数据

**4. 等级系统（Leveling System）**

```
Level 1-5: 新手
Level 6-10: 学徒
Level 11-20: 专家
Level 21-30: 大师
Level 31+: 传奇
```

每升1级，解锁新功能或权限。

### 2.5 轻量陪伴感 (Persistent Presence)

#### 【机制原理】
Duolingo 的"绿色猫头鹰"会提醒你、鼓励你、甚至"威胁"你。

#### 【迁移到选课场景】

**拟人化吉祥物：智选鸟（SmartOwl）**

**角色定位：**
- 不是"系统通知"，而是"你的选课伙伴"
- 不是"催促"，而是"关心"
- 不是"机械"，而是"有温度"

**陪伴场景设计：**

| 场景 | 时机 | 文案示例 |
|------|------|---------|
| 每日问候 | 每天首次打开 | "早上好！今天要不要看看新推荐的课程？" |
| Streak提醒 | 晚上8点（未完成） | "嘿！今天还差一个小任务哦，5分钟搞定~" |
| 成就解锁 | 完成任务时 | "哇！你刚刚解锁了新成就，快去看看！" |
| 选课季提醒 | 选课开始前1周 | "选课周要来了！你准备好了吗？" |
| 回归唤醒 | 3天未登录 | "好久不见！你的课程收藏有新评价了" |
| 学期总结 | 每学期末 | "这学期你帮助了X个同学，太棒了！" |

**Smart Owl 的多层次互动：**
- **L1 提示**：轻量通知（推送 + 角标）
- **L2 引导**：首页弹窗（今日任务）
- **L3 陪伴**：聊天式对话（"需要我帮你推荐吗？"）
- **L4 惊喜**：随机奖励（"今天运气真好，奖励你50 XP！"）

## 三、用户体验路径重构 (Experience Journey Redesign)

### 3.1 首次体验路径（First-Time User Experience, FTUE）

**设计目标：**
- 30秒内体验到价值
- 5分钟内产生首个"微成就"
- 10分钟内建立"明天还会来"的预期

**完整路径设计：**

```
第1秒：打开APP
├─ 视觉：简洁、明亮、友好
├─ 文案："嘿！准备好掌握选课技巧了吗？"
└─ CTA："开始探索" [大按钮]

第10秒：选择大学
├─ 快速定位："你的大学是？"
├─ 3个选项：Adelaide / Melbourne / Other
└─ 反馈："太好了！已为你准备好Adelaide的所有课程"

第30秒：选择目标
├─ 问题："你最想做什么？"
├─ 4个选项卡片：
│   ├─ "找到最好的教授"
│   ├─ "规划下学期课表"
│   ├─ "提升我的GPA"
│   └─ "帮助其他同学"
└─ 引导："选1个，我们马上开始"

第1分钟：首个微任务
├─ 场景：根据用户选择，分配对应任务
├─ 示例：如果选"找教授"
│   └─ 任务："浏览3门你感兴趣的课程"
│       ├─ 进度："1/3"（实时更新）
│       ├─ 引导："左滑收藏，右滑跳过"
│       └─ 完成："+10 XP，解锁Level 1！"
└─ 关键：任务极简，必定成功

第3分钟：首次社交互动
├─ 提示："看看其他同学怎么说"
├─ 展示：3条热门评价（预加载）
├─ CTA："点赞你觉得有用的评价"
└─ 反馈："你的点赞帮助了这位同学！+5 XP"

第5分钟：建立长期目标
├─ 展示：能力地图（刚刚点亮了Level 1）
├─ 文案："你刚刚迈出了第一步！"
├─ 下一步："明天继续，解锁'高级筛选'功能"
└─ CTA："设置每日提醒"（推送权限请求）

第10分钟：锚定习惯
├─ 总结："今天你完成了X个任务，获得X XP"
├─ 预告："明天的任务更有趣，还有神秘奖励哦"
├─ 社交证明："已有X位Adelaide同学在使用"
└─ 退出提示："别忘了明天回来！"
```

**为什么30秒内不会退出？**
1. **即时价值**：10秒内看到相关课程
2. **清晰引导**：每一步都有明确CTA
3. **进度可见**："1/3"让人想完成
4. **小胜利**：第一个任务必定成功 → 多巴胺 → 继续

### 3.2 每日使用路径（Daily Habit Loop）

**设计目标：**
- 每天5分钟，完成1-2个微任务
- 无需深度思考，可随时中断
- 每次都有"小胜利"感

**日常循环（Daily Hook）：**

```
触发（Trigger）
├─ 外部触发：
│   ├─ 推送："今日任务已准备好，快来完成吧！"
│   ├─ 时间：每天下午3点（课间休息时间）
│   └─ 角标：APP图标显示"1"
└─ 内部触发：
    ├─ 好奇："今天有什么任务？"
    └─ FOMO："Streak不能断！"

行动（Action）
├─ 打开APP
├─ 首页直接显示："今日任务"卡片
├─ 点击进入 → 3个微任务（5分钟内完成）
│   ├─ 任务1：快速点评（30秒）
│   ├─ 任务2：难度评估（1分钟）
│   └─ 任务3：帮助新生（2分钟）
└─ 完成 → 即时反馈

奖励（Reward）
├─ 即时奖励：
│   ├─ "+30 XP，今天的目标完成了！"
│   ├─ Streak +1："连续X天打卡"
│   └─ 随机惊喜："今天运气爆棚，双倍XP！"
├─ 可变奖励（维持新鲜感）：
│   ├─ 周一：解锁新徽章
│   ├─ 周三：获得"幸运宝箱"（随机奖励）
│   └─ 周五：社区排行榜更新
└─ 社交奖励：
    └─ "你的贡献帮助了X个同学"

投入（Investment）
├─ 数据沉淀：
│   ├─ 收藏的课程 +3
│   ├─ 评估历史 +5条
│   └─ 社区声望 +20
├─ 个性化增强：
│   └─ AI推荐更精准（基于今天的行为）
└─ 下次触发预埋：
    └─ "明天的任务更有趣，别忘了回来哦"
```

**关键设计：**
- **5分钟规则**：每日任务总时长不超过5分钟
- **可中断续做**：任务自动保存，随时离开
- **无惩罚设计**：今天没完成？没关系，明天继续

### 3.3 选课季高频路径（Course Selection Peak Season）

**场景分析：**
- 选课周是使用高峰期（Week 1, Week 13）
- 用户需求：快速对比、决策辅助、焦虑缓解

**高频路径设计：**

```
D-7天：准备阶段
├─ 推送："选课周要来了！开始规划吧"
├─ 任务："浏览你感兴趣的10门课程"
├─ 工具：课程收藏夹 + 对比工具
└─ 目标：建立"候选清单"（5-10门课）

D-3天：深度研究
├─ 推送："距离选课还有3天，时间规划好了吗？"
├─ 任务："完成3门课的深度评估"
├─ 工具：GPA模拟器 + 时间表工具
└─ 目标：确定"最终方案"（3-5门课）

D-Day：决策日
├─ 推送："选课开始了！快去抢位置"
├─ 快捷入口："一键跳转到选课系统"
├─ 实时提示："COMP1012还有X个位置"
└─ 社交分享："我选了这些课，你呢？"

D+1天：反思阶段
├─ 推送："选课结束了，分享你的经验吧"
├─ 任务："对你选的课程做个预期评估"
├─ 奖励："+50 XP，选课达人成就"
└─ 社区：发布"我的选课策略"帖子
```

**为什么这样设计？**
- **提前介入**：选课前就建立习惯
- **焦虑管理**：通过任务拆解降低决策压力
- **峰终体验**：选课后的反思强化记忆

### 3.4 中断后无痛回归路径（Re-engagement After Churn）

**设计目标：**
- 降低回归门槛
- 避免"从头再来"的挫败感
- 快速找回价值感

**回归路径设计：**

```
Day 3 未登录：
├─ 推送："你的Streak还在等你！"
├─ CTA："完成1个任务，立即恢复"
└─ 降低难度：今日任务减少到1个

Day 7 未登录：
├─ 推送："你的朋友刚刚解锁了新成就"
├─ CTA："回来看看发生了什么"
└─ 展示：个性化内容（"你收藏的课程有新评价"）

Day 14 未登录：
├─ 推送："特别任务��完成即可获得双倍XP"
├─ CTA："领取回归大礼包"
└─ 奖励：免费Streak修复 + 50 XP

Day 30 未登录：
├─ 邮件："这学期你帮助了X个同学，别让他们失望"
├─ CTA："查看你的影响力报告"
└─ 社交证明："你的评价被浏览了X次"
```

**无痛回归机制：**
- **渐进式唤醒**：从轻度提醒到重度激励
- **价值重新展示**：提醒用户过去的贡献
- **降低门槛**：回归首日任务更简单

## 四、习惯型使用模型 (Habit Formation Model)

### 4.1 日/周级别使用节奏设计

**每日节奏（Daily Rhythm）：**

| 时间段 | 场景 | 任务类型 | 时长 |
|-------|------|---------|------|
| 08:00-09:00 | 上课前 | 快速点评 | 1分钟 |
| 12:00-13:00 | 午休 | 浏览课程 | 3分钟 |

## 八、风险与应对（Risk Mitigation）

### 风险1：用户疲劳（Gamification Fatigue）

**风险描述：** 游戏化过度，用户感到被操控

**应对策略：**
- 允许"跳过任务"（不扣分）
- 提供"专注模式"（关闭所有游戏化元素）
- 定期用户访谈，调整奖励强度

### 风险2：内容质量下降

**风险描述：** 为了完成任务，用户随意提交低质量内容

**应对策略：**
- 质量审核机制（AI + 人工）
- 质量评分影响XP获取
- 高质量内容额外奖励

### 风险3：核心价值稀释

**风险描述：** 用户只关注游戏，忘记"选课"本质

**应对策略：**
- 任务设计必须与选课强相关
- 定期提醒"你的目标是什么"
- 学期节点强制回归核心价值

## 附录：关键设计原则总结

1. **碎片化优于完整性**：5分钟任务 \u003e 30分钟深度使用
2. **进步感优于信息量**："+10 XP" \u003e "查看100条评价"
3. **陪伴感优于工具感**："Smart Owl提醒你" \u003e "系统通知"
4. **习惯优于需求**："每天都来" \u003e "需要时才来"
5. **社交优于单打独斗**："帮助同学" \u003e "自己决策"
6. **奖励优于功能**："解锁新徽章" \u003e "新功能上线"
7. **游戏化优于严肃化**："Level 3选课策略家" \u003e "已提交3条评价"

---

**最终目标实现路径：**

```
用户不再问："我需要查教授评价吗？"
而是想："今天OhMyProfessors的任务完成了吗？"
甚至成为："我每天都会打开一下，已经成习惯了"
```

**这就是 Duolingo 式的成功。**
